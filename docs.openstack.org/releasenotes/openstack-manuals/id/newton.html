Rilis Newton¶

Perubahan pengguna terlihat¶

  • Installation Guide telah diubah namanya menjadi Installation Tutorial, dan sekarang termasuk layanan Container Infrastructure Management (magnum), layanan Messaging (zaqar), layanan Key Manager (barbican), dan layanan Bare Metal (ironis).

  • Command-Line Interface Reference meninggalkan referensi perintah openstack yang mendukung referensi dalam repository OpenStackClient.

  • Configuration Reference mengkonsolidasikan konfigurasi umum seperti koneksi database dan pesan RPC.

  • Configuration Reference mendukung layanan Application Catalog.

  • Glossary diorganisasi kembali untuk mendukung kemudahan akses bagi pengguna.

  • URL dan nama file diubah untuk menggunakan hanya tanda hubung (hyphen) untuk konsistensi dan optimasi mesin pencari.

perubahan internal¶

  • Installation Tutorial sekarang dapat mempublikasikan dokumentasi dari repositori proyek tertentu.

  • Operations Guide sekarang menggunakan RST sebagai format sumber. Ini melengkapi transisi ke RST, dan alat-alat DocBook lama yang telah pensiun.

  • Isi dari repositori terpisah untuk operations-guide dan ha-guide telah dipindahkan ke repositori OpenStack-manual. Repositori operations-guide dan ha-guide telah pensiun.

Panduan API¶

  • Dokumentasi referensi API telah dipindahkan dari repositori api-site ke repositori tertentu dari suatu proyek. Koleksi API docs sekarang tersedia di API Quick Start page.

  • Situs Compute API dan DNS API menawarkan contoh bagus dari referensi API baru, dikelola oleh tim proyek daripada tim docs pusat.

  • Diperbarui Contributor Guide untuk memasukkan informasi API Guides tertentu.

  • Pedoman lengkap API Documentation dengan API Working Group.

Panduan desain arsitektur¶

  • Sebuah Architecture Design Guide yang direvisi saat ini sedang dalam pengembangan, tidak ada perubahan untuk panduan saat ini.

Panduan kontributor¶

  • Terkumpul konten pengalaman pengguna dalam satu bagian.

  • Ditambahkan konten pribadi ke bagian pengalaman pengguna.

  • Diperbaharui pedoman teks antarmuka pengguna.

  • Diperbaharui saran grafis.

  • Ditambahkan informasi tentang pembuatan Installation Tutorials untuk project-specific.

Panduan ketersediaan tinggi (high availability)¶

  • Dihapus arsitektur Keepalived karena tidak lagi dianjurkan.

  • Diganti basic node dan layanan instruksi instalasi dengan link.

  • Ditambahkan bagian mengenai pemulihan instance otomatis.

  • Memberikan rincian untuk menghubungi komunitas OpenStack HA.

  • Berbagai macam pembaharuan dan pembersihan.

Petunjuk instal¶

  • Berganti nama menjadi Installation Tutorial.

  • Installation Tutorial sekarang dapat mempublikasikan dokumentasi dari repositori proyek tertentu.

  • Diterbitkan panduan proyek tertentu untuk layanan Container Infrastructure Management (magnum), layanan Messaging (zaqar), layanan Key Manager (barbican), dan layanan Bare Metal (ironis).

  • Dibuat alat cookiecutter untuk memastikan panduan proyek spesifik yang memiliki struktur yang konsisten.

Panduan Networking¶

  • Hasil restrukturisasi lengkap sebagai panduan.

  • Organisasi ulang dan menulis ulang dimana contoh penyebaran menggunakan “building blocks” untuk meningkatkan kegunaan dan pemahaman. Setelah pemilihan salah satu Linux bridge atau Open vSwitch mechanism driver, audience dapat meningkatkan pengerahan arsitektur kompleks yang membangun contoh lebih sederhana dari sebelumnya. Juga mengurangi duplikasi konten.

  • Ditambahkan subnet layanan.

  • Ditambahkan service function chaining (sfc)

  • Ditambahkan RBAC.

  • Ditambahkan kolam subnet

  • Ditambahkan lingkup alamat (address scope)

  • Ditambahkan routing dinamis BGP.

  • Ditambahkan DPDK untuk Open vSwitch.

  • Ditingkatkan kelompok keamanan (security group).

  • ditingkatkan cakupan alamat (address scope)

  • Ditingkatkan SR-IOV.

  • Peningkatan resolusi DNS.

  • Peningkatan high-availability untuk DHCP.

Panduan operasi¶

  • Completed RST conversion.

  • Ditambahkan dokumentasi operasi perusahaan termasuk informasi tip menatasi masalah RabbitMQ, petunjuk untuk mengambil (retrieve) alamat IP yang hilang, dan prosedur untuk mengelola alamat IP mengambang antar instance.

  • Telah dihapus instruksi untuk menginstal DevStack dan informasi kontribusi untuk OpenStack, dimana keduanya didokumentasikan di situs proyek DevStack dan Infrastructure Manual masing-masing.

  • Diperbaharui perintah klien command-line OpenStack ke perintah OpenStackClient.

Security Guide¶

  • Ditembahkan OSSNs 0063, 0066, 0068, 0069, 0073.

  • Bagian baru pada lalu lintas rate-limiting dan API endpoint.

  • Diganti contoh Keystone CLI dengan contoh OpenStack CLI.

  • Perubahan yang dapat dibaca untuk link, image, dan format dokumen.

  • Memperbaharui bagian kepatuhan (compliance) sekitar tahapan audit dan link.

  • Contoh dimigrasikan dari contoh pemuatan drive langsung layanan Identity (keystone) yang telah usang ke stevedore (pemuat kuli pelabuhan)

  • Diperbaharui contoh perintah untuk ketepatan dan kesetaraan dengan versi terbaru.

  • Ditambahkan alat pembangunan sehingga Security Guide dapat dibangun dengan bagian OSSN dan TA secara bersamaan ataupun secara terpisah.

  • Diperbaharui istilah glossary untuk Nginx, SPICE, Data Loss Prevention (DLP), Trusted Platform Module (TPM), Secure Boot, dan akronim lainnya.

Panduan pelatihan¶

  • Perbaikan dan restrukturisasi Upstream Training

  • Ditambahkan bab baru dalam draft Training guides

Laboratorium pelatihan¶

  • Laboratorium pelatihan landing page diterbitkan di bawah docs.openstack.org:

    • Pengguna dapat men-download file zip/tar untuk platform Windows, Linux dan Mac OS X untuk rilis yang didukung.

  • Fitur PXE untuk training-labs (pelatihan-laboratorium):

    • Fungsi booting PXE tersedia sebagai pluggable driver secara paralel dengan KVM/VirtualBox.

    • Boot PXE juga bisa digunakan sebagai mekanisme untuk penyediaan KVM/VirtualBox berdasarkan beban kerja.

  • perbaikan stabilitas:

    • Berbagai perbaikan stabilitas telah ditambahkan dalam rilis ini.

    • Dukungan platform Windows telah diperbarui dan diperbaiki.

    • Klaster pelatihan harus memiliki tambahan tingkat pengerahan yang lebih tinggi.

    • Perbaikan jaringan yang terkait dengan tantangan.

  • Perbaikan kinerja:

    • Kecepatan penyiapan kluster membaik setelah penambahan beberapa fitur.

    • Backend KVM/Libvirt sekarang menggunakan penyimpanan bersama. Hard disk untuk cluster yang diffs di atas dasar disk. Ini menggunakan ruang penyimpanan berkurang.

    • Block storage (cinder) memiliki disk dedikasi sebagai lawan dari menjadi file yang di-mount sebagai perangkat loop. Disk dediksi ini harus meningkatkan kinerja dan memberikan keeratan (persisten) untuk cinder-volume di reboot.

    • Peningkatan kinerja dengan memperbaiki terjadi kondisi bergegas karena berlebihan (overhead) virtualisasi.

    • Perbaikan skrip perpustakaan.

  • Cakupan uji:

    • Ditambahkan lebih banyak tes untuk skrip repeat_test.

    • Kasus pengujian antarmuka user untuk memeriksa ketersediaan horizon telah dilaksanakan.

  • Pembaruan untuk mendukung sistem operasi baru. Misalnya, Ubuntu 16.04.

  • Menambahkan dukungan untuk arsitektur x86/i386.

  • Update ke CLI bersama dengan banyak perbaikan untuk mengatasi fitur baru.

  • Dukungan Newton:

    • Dukungan Newton harus segera tersedia setelah rilis OpenStack.

    • Penundaan ini disengaja, untuk membiarkan paket distribusi stabil, dan untuk memastikan semua kasus uji (test case) dan pemeriksaan keandalan memenuhi kriteria tertentu.

Panduan pengguna¶

  • Ditambahkan informasi replikasi Redis dan informasi konfigurasi Rootwrap Daemon ke Administrator Guide.

  • Organisasi ulang bab Administrator Guide Telemetry untuk meningkatkan arsitektur informasi.

  • Dimirgasikan konten tentang konfigurasi dan menyesuaikan Dashboard, dimana sebelumnya dipelihara di Configuration Reference, sekarang ke bab Dashboard dari Administrator Guide.

Newton Release¶

Für Benutzer sichtbare Änderungen¶

  • Die Installationsanleitung wurde in Installationstutorial umbenannt und enthält jetzt Container Infrastrukture Management Dient (magnum), Nachrichtendienst (zaqar), Schlüsselmanager Dienst (barbican) und Bare Metal Dienst (ironic).

  • Die Kommandozeilen-Interface Referenz vermeidet die openstack Kommandoreferenz zugunsten der Referenz im OpenStackClient Repository.

  • Konfigurationsreferenz konsolidiert die allgemeine Konfiguration wie z.B. Datenbankverbindungen und RPC Messaging.

  • Konfigurationsreferenz unterstützt den Applikationskatalog-Dienst.

  • Glossar neu organisiert, um einfacheren Zugriff für Benutzer zu ermöglichen.

  • URL und Dateinamen geändert, so dass aus Gründen der Konsistenz und Optimierung der Suchmaschine nur Bindestriche verwendet werden.

Interne Änderungen¶

  • Das Installationstutorium kann jetzt Dokumentation aus projektspezifischen Repositories veröffentlichen.

  • Der Operations Guide verwendet jetzt RST als Quellformat. Somit ist der Überhang zu RST abgeschlossen und die alten DocBook Werkzeuge werden nicht mehr benötigt.

  • Der Inhalt der separaten Repositories operations-guide und ha-guide wurde ins openstack-manuals Repository verschoben. Die operations-guide und ha-guide repositories wurden stillgelegt.

API-Anleitungen¶

  • Die API Referenzdokumentation wurde vom api-site Repository in projektspezifische Repositories verschoben. Die Sammlung der API Docs its jetzt über die API Quick Start page <http://developer.openstack.org/api-guide/quick-start/> verfügbar_.

  • Die Compute API und DNS API Sites bieten gute Beispiele zur neuen API Referenz. Sie werden von Projektteam und nicht mehr von einem zentralen Docs Team gewartet.

  • Aktualisiert wurde der Contributor Guide mit bestimmten API Guides Informationen.

  • Abgeschlossen wurde API Dokumentation Richtlinien mit der API Arbeitsgruppe.

Architektur Design Guide¶

  • Eine überarbeiteter Architektur Design Guide ist derzeit in Arbeit, keine Änderungen am aktuellen Guide.

Contributor Guide¶

  • User Experience Inhalte in einer einzigen Sektion zusammengefasst.

  • Inhalt zu Personas zur User Experience Sektion hinzugefügt.

  • User Interface Text Richtlinien wurden überarbeitet.

  • Grafik Advice überarbeitet.

  • Informationen zum erstellen projektspezifischer Tutorials hinzugefügt.

High Availability Anleitung¶

  • Keepalived Architektur entfernt, da sie nicht mehr gefördert wird.

  • Grundsätzliche Anweisungen zur Knoten- und Diensteinstallation durch Links ersetzt.

  • Sektion zur automatischen Instanzwiederherstellung hinzugefügt.

  • Details zum kontaktieren der OpenStack HA Community angegeben.

  • Gruppierte Aktualisierungen und Bereinigungen.

Installationsanleitung¶

  • Umbenannt nach Installationstutorium.

  • Das Installationstutorium kann jetzt Dokumentation aus projektspezifischen Repositories veröffentlichen.

  • Projektspezifische Anleitungen für Container Infrastrukture Management Dient (magnum), Nachrichtendienst (zaqar), Schlüsselmanager Dienst (barbican) und Bare Metal Dienst (ironic) hinzugefügt.

  • Erstepllung des cookiecutter Werkzeugs zum sicherstellen, dass projektspezifische Anleitungen eine konsistente Struktur erhalten.

Netzwerkanleitung¶

  • Anleitung komplett neu strukturiert.

  • Bereitstellungsbeispiele wurden neu geschrieben und als “building blocks” organisiert, um die Verwendbarkeit und das Verständnis zu verbessern. Nach der Auswahl von Linuxbridge oder Open vSwitch kann das Publikum zunehmend komplexe Architekturen bereitstellen, aufbauend auf zuvor einfacheren Beispielen. Doppelte Inhalte entfernt.

  • Dienst Subnetz hinzugefügt.

  • Dienst Funktionsverkettung hunzugefügt.

  • RBAC hinzugefügt.

  • Subnetzpools hinzugefügt.

  • Adressbereiche hinzugefügt.

  • BGP dynamisches Routing hinzugefügt.

  • DPDK für Open vSwitch hinzugefügt.

  • Sicherheitsgruppen verbessert.

  • Adressbereiche verbessert.

  • SR-IOV verbessert.

  • DNS_Auflösung verbessert.

  • Hochverfügbarkeit für DHCP verbessert.

Betriebsanleitung¶

  • RST-Konvertierung abgeschlossen.

  • Enterprise Operations Dokumentation einschließlich Informationen zur RabbitMQ Fehlersuche hinzugefügt, sowie Anweisungen zum Abruf verlorener IP-Adressen und Prozeduren zur Verwaltung von Floating IP-Adressen zwischen Instanzen.

  • Anweisungen zu Installation von DevStack und Mitarbeit in OpenStack entfernt, da dies auf der DevStack Projektwebseite bzw. im Infrastruktur Manual dokumentiert ist.

  • Aktualisieren der OpenStack command-line Client Kommandos zu OpenStackClient Kommandos.

Sicherheitsanleitung¶

  • OSSNs 0063, 0066, 0068, 0069, 0073 hinzugefügt.

  • Neue Sektion zur Durchsatzbegrenzung und API Endpunkt-Verkehr.

  • Keystone CLI Beispiele wurden mit OpenStack CLI Beispielen ersetzt.

  • Verbesserung der Lesbarkeit von Verknüpfungen, Bildern und des Dokumentenformats.

  • Aktualisierungen der Compliance Sektion über Auditphasen und Links

  • Beispiele von veralteten Identitätsdienst (keystone) direct driver loading nach stevedore verschoben.

  • Kommandobeispiele für verbesserte Genauigkeit und um auf dem gleichen Stand wie die letzte Version zu sein aktualisiert

  • Bauwerkzeuge hinzugefügt, so dass der Security Guide separat und mit den OSSN und TA Sektionen gebaut werden kann.

  • Glossar-Ausdrücke für Nginx, SPICE, Data Loss Prevention (DLP), Trusted Platform Module (TPM), Secure Boot und andere Akronyme aktualisiert..

Trainingsanleitungen¶

  • Verbesserung und Neustrukturierung des Upstream Trainings

  • Neue Kapitel zum Entwurf `Training Guides <http://docs.openstack.org/draft/training-guides/>`hinzugefügt _

Training Labs¶

  • Training Labs Startseite unter docs.openstack.org veröffentlicht:

    • Anwender können zip/tar Dateien für Windows, Linux und Mac OS X Plattformen für unterstütze Releases herunterladen.

  • PXE Feature für training-labs:

    • PXE Bootfunktionalität ist als steckbarer Treiber parallel für KVM/VirtualBox verfügbar.

    • PXE Boot konnte ebenso als Mechanismus zur Provisionierung von KVM/VirtualBox basierten Workloads verwendet werden.

  • Stabilitätsverbesserungen:

    • Mannigfaltige Stabilitätsverbesserungen wurden in diesem Release hinzugefügt.

    • Unterstützung der Windows-Plattform wurde aktualisiert und verbessert.

    • Trainingcluster sollte außerdem häufiger bereitgestellt werden.

    • Verbesserungen für netzwerkbezogene Schwierigkeiten.

  • Leistungsverbesserungen:

    • Geschwindigkeit der Clustereinrichtung wurde nach Hinzufügen einer Vielzahl von Features verbessert.

    • KVM/Libvirt Backends verwenden jetzt Shared Storage. Festplatten für den Cluster sind diffs auf den Basisdisks. Dies verwendet weniger Storage-Speicher.

    • Block Storage (cinder) hat fest zugewiesene Platten, im Gegensatz zu als Loop-Device gemounteten Dateien. Dies verbessert die Performanz und bietet Persistenz für cinder-Datenträger über Reboots.

    • Performanzverbesserungen durch Beseitigung von Fehlern (race conditions) aufgrund von Virtualisierungsüberladungen.

    • Verbesserungen an den Bibliothek-Skripten.

  • Testabdeckung:

    • Mehr Tests zu repeat_test Skripts hinzugefügt.

    • Userinterface Tests zur Verfügbarkeitsprüfung von Horizon wurden hinzugefügt.

  • Aktualisierungen zur Unterstützung neuer Betriebssysteme, z.B. Ubuntu 16.04.

  • Unterstützung fürx86/i386 Architektur hinzugefügt.

  • Aktualisierung des CLI zusammen mit vielen Verbesserungen und neuen Eigenschaften.

  • Newton Unterstützung:

    • Newton Unterstützung ist geplant als verfügbar kurz nach dem dem OpenStack Release.

    • Diese Verzögerung ist beabsichtigt, um die Distributionspakete zu stabilisieren und sicherzustellen, dass alle Testfälle und Verlässlichkeitsprüfungen bestimmte Kriterien erfüllen.

Benutzeranleitungen¶

  • Redis Replikationsinformationen und Rootwrap Konfigurationsinformationen zum Administrator Guide hinzugefügt.

  • Telemerie-Kapitel im Administrator Guide neu organisiert, um die Informationsarchitektur zu verbessern.

  • Zuvor in der Konfigurationsreferenz enthaltene Inhalte zur Konfiguration und Anpassung des Dashboards wurden in das Dashboard-Kapitel im Administrator Guide migriert.

Übersetzungen¶

Neben Aktualisierungen der vorhandenen übersetzten Manuals hat das Internationalisierungsteam die folgenden neuen Manuals hinzugefügt:

  • Indonesisch

    • API Anleitung veröffentlicht.

    • Installations-Tutorium für Mitaka veröffentlicht.

    • Netzwerkanleitung veröffentlicht.

  • Intalienisch

    • Upstream-Training veröffentlicht.

  • Japanisch

    • Installations-Tutorium für Mitaka veröffentlicht.

  • Koreanisch

    • Intallations-Tutorien für Mitaka und Liberty veröffentlicht.

  • Vereinfachtes Chinesisch

    • API Anleitung veröffentlicht.

The cinder.volume.drivers.ibm.flashsystem_iscsi Module¶

Volume driver for IBM FlashSystem storage systems with iSCSI protocol.

Limitations: 1. Cinder driver only works when open_access_enabled=off.

do_setup(ctxt)¶

Check that we have all configuration details from the storage.

initialize_connection(*args, **kwargs)¶

Perform work so that an iSCSI connection can be made.

To be able to create an iSCSI connection from a given host to a volume, we must: 1. Translate the given iSCSI name to a host name 2. Create new host on the storage system if it does not yet exist 3. Map the volume to the host if it is not already done 4. Return the connection information for relevant nodes (in the proper I/O group)

terminate_connection(*args, **kwargs)¶

Cleanup after connection has been terminated.

When we clean up a terminated connection between a given connector and volume, we: 1. Translate the given connector to a host name 2. Remove the volume-to-host mapping if it exists 3. Delete the host if it has no more mappings (hosts are created automatically by this driver when mappings are created)

validate_connector(connector)¶

Check connector for enabled protocol.
